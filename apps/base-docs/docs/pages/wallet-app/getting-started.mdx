# Getting Started with Mini Apps

## Overview

Mini Apps are lightweight web applications that run natively within clients like Coinbase Wallet. They provide instant access without requiring downloads, offer seamless wallet integration, and enable direct distribution through social feeds, creating effective viral loops for user acquisition and engagement.

## What Mini Apps Are (Technical Definition)

A Mini App is composed of:

- Your existing web application
- `<MiniKitProvider>` wrapper component
- `farcaster.json` manifest file
- Standard deployment to any hosting platform (Vercel, Netlify, etc.)

No rebuilding, architectural changes, or complex integrations are required.

## Code Implementation

### Before (Standard Web Application)

```jsx
function App() {
  return (
    <div>
      <MyExistingComponent />
      <AnotherComponent />
    </div>
  );
}
```

### After (Mini App Implementation)

```jsx
import { MiniKitProvider } from '@coinbase/onchainkit/minikit';

function App() {
  return (
    <MiniKitProvider projectId="your-id">
      <MyExistingComponent />  {/* Existing components unchanged */}
      <AnotherComponent />     {/* Existing components unchanged */}
    </MiniKitProvider>
  );
}
```

**Changes Required**: Two import lines and one wrapper component. All existing components remain unchanged.

### Account Association (Domain Verification)

Domain ownership verification is required. Generate your manifest using:

**MiniKit Method:**

```shell
npx create-onchain --manifest
```

**Alternative Farcaster Tool Process:**

You can use the [Farcaster manifest tool](https://farcaster.xyz/~/developers/mini-apps/manifest) for generation.

### Image Requirements

- **iconUrl**: 200x200px PNG/JPG format
- **splashImageUrl**: 200x200px PNG/JPG (displayed during app loading)
- **heroImageUrl**: 1200x628px PNG/JPG (for featured placement)
- **screenshotUrls**: App store screenshots in various sizes

## Deployment Process

The deployment process remains identical to standard web applications:

1. **Build Application**: `npm run build` (standard process)
2. **Add Configuration**: `public/.well-known/farcaster.json`
3. **Deploy**: Use any hosting platform (Vercel, Netlify, custom servers)
4. **Browser Testing**: Functions as standard web application
5. **Farcaster Testing**: You can test using the farcaster tools [here](https://farcaster.xyz/~/developers/mini-apps/manifest).

No special hosting, new infrastructure, or complex setup procedures required.

## Frequently Asked Questions

### Application Rebuild Requirements

**Q: Do we need to rebuild our entire application?**
**A:** No. Wrap your existing application in `<MiniKitProvider>`. Implementation complete.

### Current Application Impact

**Q: Will this break our current web application?**
**A:** No. Mini Apps function in regular browsers. The same codebase works across all platforms.

### Blockchain Expertise Requirements

**Q: Do we need blockchain expertise?**
**A:** No. Integration complexity is similar to Stripe implementation.

### User Data and Privacy

**Q: What about user data and privacy?**
**A:** Maintain current practices. You control data, backend systems, and privacy policies.

### Performance Impact

**Q: What is the performance impact?**
**A:** Minimal impact. MiniKit SDK is approximately 50KB. Application performance remains unchanged.

### Non-React Framework Support

**Q: We don't use React/Next.js. Is this compatible?**
**A:** Yes. The Farcaster SDK supports Vue, Angular, Svelte, and any web framework.

## Implementation Examples

### Simple Example (15-minute implementation)

```shell
npx create-onchain --mini
cd my-miniapp
npm run dev
```

### Non-React Implementation (Vue, Svelte, Vanilla JS)

```shell

cd my-miniapp
npm run dev
```

**Result**: Functional Mini App with wallet integration, transaction capabilities, and social features.
